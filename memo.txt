1117
	멀티플레이
	-네트워크 : 방대하고 복잡

	네트워크를 통해 데이터를 전달하는 기본적 방법
	네트워크 지원 때문에 달라져야 하는 게임 설계 방법

	Multi User Dungeon
		96년: 바람의 나라
		97년: 리니지
		온라인 게임

	프로토콜(Protocol)≒보내는 규칙
		패킷(Packet)≒주소(헤더)
		페이로드(Payload)≒편지
		네트워크를 통해 데이터를 전송하는 데 필요한 규칙
		패킷의 구성 방법과 패킷을 전송할 때 해야 할 일을 정의

	IP: Internet Protocol
		인터넷을 통해 데이터를 전달하려면 반드시 따라야 하는 프로토콜

	IP의 헤더
		헤더(20byte)
			버전, 길이, 서비스, 패킷 길이

	IPv4, IPv6

	ICMP: Internet Control Message Protocol
		Echo(에코)
			에코 요청 => 에코 응답 : 지정한 주소에 패킷을 보낸 뒤에 즉시 되돌려 받을 수 있음
			패킷이 두 컴퓨터 사이를 왕복하는 시간을 측정할 때 사용 가능

	TCP: 전송 제어 프로토콜
		게임에서 네트워크를 통해 데이터 전송 시 사용하는 가장 대표적 두가지 방법 중 하나
		모든 패킷의 전송 순서 보장
		연결 중심 프로토콜 : 두 컴퓨터가 데이터를 주고 받으려면 연결을 맺어야 함
			Hand Shaking: 연결을 맺는 데에 필요한 과정
				연결 전송 후 확인 응답

		게임에서: 전송 순서 보장이 불필요한 병목 현상 일으킬 때가 많음

		유용한점
			모든 네트워크는 패킷의 크기를 제한하는 최대 전송 단위를 가짐
			OS단에서 알아서 적절한 크기의 패킷으로 나눌 수 있음

	턴 방식의 게임
		네트워크를 통해 오고가는 모든 정보는 플레이어가 자신의 턴에 수행한 동작 간에 서로 밀접한 관련이 있을 확률 높음
	MMO 게임
		WOW의 경우 모든 데이터 전송에 TCP 방식 사용
		WOW에서의 모든 행동은 철저하게 순서가 정해져 있음
		=>순서 보장 프로토콜 사용하는 것이 마땅

	포트(Port)
		대부분의 웹 서비스: 80번 포트
		65000

	UDP: User Datagram Protocol
		비연결 방식
		신뢰성 없는 프로토콜
		TCP와 포트가 별개 => 동일 포트 사용해도 충돌하지 않음
		FPS, 액션, RTS에서 주로 사용

	네트워크 토폴로지
		네트워크 게임 세션 내의 여러 컴퓨터를 연결하는 방식
		서버/클라이언트(Server/Client)
			보안/해킹: 클라이언트의 동작을 서버가 확인
			서버가 모든 클라이언트의 최신 정보를 가지고 있기 때문
			싱글 플레이를 특별한 형태의 멀티 플레이처럼 구현
		피어 투 피어(Peer to Peer)
			스타2 등의 RTS에서 사용
			150~200ms 정도 턴 단위로 네트워크 업데이트하는 모델에서 주로 사용
			네트워크를 통해 오가는 정보: 플레이어가 입력한 명령
